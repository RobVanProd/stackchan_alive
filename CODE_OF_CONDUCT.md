# Stackchan: Alive Community Code Of Conduct

## Our Commitment

We want Stackchan: Alive to be a welcoming project for people learning about robotics, embedded
systems, local AI, accessibility, art, and character design. Participants are expected to treat
each other with patience, respect, and good faith regardless of experience, identity, background,
or role.

## Expected Behavior

- Be constructive, specific, and kind in issues, reviews, discussions, and test reports.
- Respect physical-safety stops, privacy boundaries, licensing decisions, and evidence gates.
- Separate observed behavior from hypotheses and correct mistakes openly.
- Give credit for ideas, code, designs, testing, and documentation.
- Make room for newcomers to ask questions without ridicule.
- Accept a maintainer decision to pause unsafe testing or remove sensitive material.

## Unacceptable Behavior

- Harassment, threats, discrimination, sexualized attention, personal attacks, or sustained
  disruption.
- Publishing another person's private information, recordings, images, credentials, or identity
  data without clear permission.
- Pressuring someone to run unsafe actuator, electrical, audio, lighting, camera, or power tests.
- Knowingly submitting unlicensed voice, model, media, or code assets.
- Misrepresenting synthetic, short-run, or different-image evidence as physical release proof.
- Using project channels to distribute malware, secrets, exploits, or restricted artifacts.

## Scope

This policy applies in repository spaces and when representing the project in public. Technical
disagreement is welcome; hostility toward people is not.

## Reporting And Enforcement

For ordinary moderation concerns, contact the maintainers through the repository owner's GitHub
profile contact methods and avoid a public issue when doing so would expose a person or private
material. Use [SECURITY.md](SECURITY.md) for security or privacy vulnerabilities. Do not place
credentials, private recordings, camera frames, or identifying evidence in a public report.

Maintainers may edit or remove content, close discussions, reject contributions, issue warnings,
or temporarily or permanently restrict participation when behavior conflicts with this policy.
Enforcement decisions should consider context, impact, safety, and a participant's willingness to
repair harm. Good-faith reports must not be retaliated against.
